## 1. 실행 방법

모든 파일, 폴더를 다운 후 하나의 폴더에 저장합니다.
index.html을 직접 열면 `키워드 검색` 기능은 확인할 수 있지만, `AI 채팅`은 브라우저 보안 정책 때문에 로컬 서버 실행이 필요합니다.


### AI 채팅 모드 실행 준비

홈 검색은 `키워드 검색`과 `AI 채팅` 두 방식으로 전환할 수 있습니다. `AI 채팅`은 사용자의 PC에서 실행되는 Ollama와 `gemma3:1b` 모델이 필요합니다.

```powershell
& "$env:LOCALAPPDATA\Programs\Ollama\ollama.exe" pull gemma3:1b
python -m http.server 5500 --bind 127.0.0.1
```

브라우저에서 아래 주소로 접속한 뒤 홈 화면의 `AI 채팅`을 선택합니다.

```text
http://127.0.0.1:5500
```

Ollama가 설치되지 않았거나 모델이 준비되지 않은 환경에서도 `키워드 검색` 방식은 그대로 사용할 수 있습니다.
# 2. 각 탭 설명
## 홈

시작 화면

- 검색에 예시처럼 `파이썬 챗봇`, `웹 포트폴리오`, `게임`, `웹 보안` 등을 입력
- 추천 방식 전환 : 기존 JavaScript 키워드 검색 또는 로컬 AI 기반 채팅 답변 선택
- Windows, macOS, Linux 중 현재 OS 환경 선택
- 필터 : 무료/유료, 개발 분야, 사용 언어 조건 설정
- 입력 결과 : 개발앱 추천과 AI 추천 각 3개씩 표시
- 추천 카드 : 대표 아이콘, 비용 정보, 간단 설명, 추천 버전, 다운로드 버튼이 표시 (다운로드 버튼 클릭 시 다운로드 페이지로 이동)
- 체크리스트에서 체크한 도구 = 다운로드 버튼 대신 세팅 완료 표시
- 기타 : 개발 분야 목록, 개발자 소통 링크, VS Code 단축키 표

## 도구 모음

개발 프로그램과 AI 도구 : 단계별, 전체보기 방식

- 단계별 보기 : 1단계, 2단계, 3단계 개발환경 선택
- 필터 : 분야, 언어, 도구 종류 조건 설정
- Flicking 플러그인 : 도구 카드를 페이지 단위로 표시
- AI 비교 : 두 AI 도구의 이름, 특징, 비용을 표로 비교
- 도구 카드 : 카드에 마우스 올리면 한 줄 설명 확인 가능 (클릭 시 다운로드 페이지로 이동)

## 환경

개발환경 세팅 상태, 사용 가능한 언어 수준을 관리

- 개발환경 세팅 체크리스트 : 카테고리별 도구 설치 여부를 체크
- 체크된 도구 : 홈 추천 카드에서 세팅 완료 상태로 반영
- 사용 가능 언어 체크리스트 : C, C++, C#, Java, Kotlin, Python, HTML/CSS/JavaScript, TypeScript, SQL 등
- 기초, 중급, 심화 단계로 수준 설정
- 로그인한 사용자별 체크리스트 정보 로컬 저장소에 분리 저장

## AI 프롬프트

프롬프트 작성 / 보관

- 프롬프트 : 제목 / 내용 저장
- 저장된 프롬프트 : 왼쪽 보관함에 표시
- 보관함 : 항목을 클릭 = 기존 내용을 불러와 수정 가능
- X 버튼 : 저장된 프롬프트 삭제
- 텍스트 프롬프트 구조, 컨텍스트 엔지니어링 작성 가이드 제공
- 개발 질문 템플릿과 컨텍스트 패키지 템플릿 수정 가능

## 학습 일지

공부한 내용 기록

- 제목, 학습 분야, 학습 언어, 학습 내용, 사용한 AI 저장
- 저장된 기록 : 카드 형태로 표시
- 카드 : 클릭하면 분야, 언어, 사용 AI, 학습 내용 확인 가능
- 필터 : 검색어, 학습 분야, 학습 언어 조건으로 기록 탐색
- 사용자별 학습 기록 로컬 저장소 분리 저장

주요 기능 함수

- 프롬프트 보관함 렌더링

AI 채팅 기능

- 주제 범위 판별 및 주제 밖 질문 안내

### `js/app-init.js`

- 로그인/로그아웃 이벤트

- 프롬프트 저장/삭제 이벤트

- 창 크기 변경 시 Flicking 레이아웃 갱신

# 4. 평가 항목에 포함되는 내용

## CSS3를 사용한 수평 정렬

- 헤더, 탭, 다크모드 토글, 로그인 버튼 : Flexbox로 수평 정렬
- 홈의 OS 버튼, 필터 버튼
- 도구 카드, 체크리스트 항목 Grid/Flex 기반 정렬

## CSS3를 사용한 중앙 정렬

- 홈의 검색 입력 박스 화면 중앙 배치
- 버튼, 카드 내부 요소, Flicking 페이지 표시점 등 중앙 정렬
- 모바일 화면에서 탭 메뉴, 주요 버튼 가운데 정렬

## CSS3를 사용한 One True 레이아웃 구성

- 상단 헤더, 본문 탭 영역, 푸터 기본 페이지 구조
- 탭별 콘텐츠 : 하나의 메인 영역에서 전환
- 도구 모음 : 왼쪽 보기/필터 영역, 오른쪽 카드 영역 구성

## CSS3 절대좌표 활용

- 다크모드 토글의 원형 버튼, 해/달 아이콘 배치 위치 조정
- 일부 보조 UI 요소, 시각적 상태 표시 CSS 위치 속성으로 조정

## 반응형 웹 구현

- `css/responsive.css` : 모바일 화면 대응 미디어 쿼리 적용
- 개발 분야 목록 : 가로 스크롤 방식
- 도구 모음 카드 : 화면 크기에 따라 페이지당 표시 개수 변화
- 입력 폼, 필터, 카드 목록 : 좁은 화면에서도 겹치지 않도록 배치

## JavaScript, 객체, DOM 사용

- `appData` 객체 : 현재 사용자, 선택 OS, 검색 상태, 저장 기록, 체크리스트 관리
- JavaScript 함수 : 추천 점수 계산, 필터링, 렌더링, 저장 처리 수행
- DOM 조작 : 추천 카드, 도구 카드, 체크리스트, 프롬프트 목록, 학습 기록 목록 동적 생성

## jQuery 라이브러리 사용

- `$()` 선택자와 이벤트 처리로 탭 전환, 버튼 클릭, 입력 변경, 폼 제출 구현
- `.on()`, `.val()`, `.text()`, `.append()`, `.prop()`, `.addClass()`, `.removeClass()` 활용
- jQuery Easing 사용해 맨 위로 이동 시 부드러운 스크롤 효과

## jQuery 플러그인 사용

- Select2: select 입력창 UI 개선
- jQuery Validation: 로그인, 회원가입, 프롬프트, 학습 일지 입력 검사
- Flicking: 개발 분야 목록, 도구 카드 페이지 넘김 기능
